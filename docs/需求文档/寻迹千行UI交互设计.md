## 全局设计规范

### 品牌色调

| 用途 | 建议 |
|------|------|
| 主色调 | 旅游/户外感，建议暖色系 |
| 辅助色 | 状态颜色见下方 |
| 背景色 | 浅灰底 #F5F5F5 |

### 通用状态颜色

| 颜色标识 | 用途 |
|---------|------|
| 🔴 红色 | 待支付、待处理、紧急状态 |
| 🟡 黄色 | 待确认、跟进中、审核中 |
| 🔵 蓝色 | 待出行、进行中 |
| 🟢 绿色 | 已完成、已通过、可用 |
| 🟠 橙色 | 退款中、处理中 |
| ⚫ 灰色 | 已取消、已关闭、已过期 |

### 通用交互模式

| 模式 | 说明 |
|------|------|
| 列表加载 | 下拉加载更多，每页10-20条 |
| Toast提示 | 操作反馈，显示1.5秒自动消失 |
| 二次确认 | 删除、取消等破坏性操作需弹窗确认 |
| 空状态 | 列表为空时显示插图 + 引导文案 + 操作按钮 |
| 加载状态 | 页面级Loading骨架屏 |

### 日期状态说明

| 状态 | 显示样式 | 可点击 |
|------|---------|--------|
| 可选 | 显示价格（黑色） | ✅ |
| 选中 | 价格高亮 + 背景色 | ✅ |
| 售罄 | 显示「售罄」灰色 | ❌ |
| 过期 | 灰色不显示 | ❌ |
| 未配置 | 显示「-」或不显示 | ❌ |

### 人数选择说明

| 元素 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 成人数量 | 1 | 1 ~ 库存上限 | 至少1人 |
| 儿童数量 | 0 | 0 ~ 成人数量 | 儿童数不能超过成人数 |
| 合计金额 | - | - | 实时计算：成人价×成人数 + 儿童价×儿童数 |

### 交互逻辑

- 选择日期后，根据当日价格日历显示成人价和儿童价
- 调整人数时实时更新合计金额
- 总人数不能超过当日剩余库存，超出时提示「库存不足」
- 点击「立即预订」时校验：已选套餐、已选日期、成人数≥1
